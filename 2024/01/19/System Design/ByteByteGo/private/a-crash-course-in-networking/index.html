A Crash Course in Networking

인터넷의 핵심은 표준화된 통신 프로토콜을 사용해 데이터 교환을 용이하게 하는 상호 연결된 컴퓨터 네트워크 덕분이다. 이를 통해 모든 종류의 디바이스에서 모든 종류의 데이터를 교환할 수 있다.

이 포스트에서는 인터넷 핵심 구성 요소인 네트워크 엣지, 액세스 네트워크, 네트워크 코어, 네트워크 프로토콜, 인터넷 프로토콜 스택을 살펴봄으로써 인터넷의 본질에 대해 알아본다. 패킷 스위칭, 포워딩, 라우팅이 어떻게 작동하는지 살펴보고, 네트워크 프로토콜이 어떤 역할을 하는지 알아본다.

Internet Evolution

다음은 오늘날 우리가 알고 있는 디지털 환경을 형성한 인터넷 역사의 주요 변혁이다.

Components of the Internet

인터넷은 복잡하고 상호 연결된 네트워크로 몇 가지 주요 구성 요소로 이루어져 있다.

The Network Edge

인터넷에 연결된 컴퓨터 및 기타 장치를 흔히 엔드 시스템이라고 한다. 이런 장치는 인터넷의 가장자리에 위치하기 때문에 엔드 시스템이라 불린다.

인터넷의 최종 시스템에는 데스크톱 컴퓨터, 서버, 모바일 디바이스가 포함되며, 스마트 가전제품이나 IoT 디바이스와 같은 전통적인 방식이 아닌 “사물”이 인터넷에 최종 시스템으로 연결되는 경우가 점점 더 많아지고 있다.

엔드 시스템은 웹 브라우저, 웹 서버, 이메일 클라이언트 또는 이메일 서버와 같은 애플리케이션 프로그램을 호스팅하기 때문에 호스트라고도 한다. 호스트는 클라이언트와 서버의 두 가지 범주로 더 세분화되기도 한다. 비공식적으로 클라이언트는 데스크톱 및 모바일 PC, 스마트폰 및 이와 유사한 개인 컴퓨팅 장치인 반면, 서버는 웹 페이지 저장 및 배포, 비디오 스트리밍, 이메일 중계 및 이와 유사한 서비스를 제공하는 보다 강력한 컴퓨터다. 오늘날 검색 결과, 이메일, 웹 페이지, 동영상을 제공하는 대부분의 서버는 대규모 데이터 센터에 있다.

Access networks

“네트워크의 엣지”에 있는 애플리케이션과 엔드 시스템을 살펴보았으니 이제 엔드 시스템에서 다른 멀리 떨어진 엔드 시스템으로 가는 경로에서 엔드 시스템을 첫 번째 라우터(엣지 라우터)에 물리적으로 연결하는 네트워크인 액세스 네트워크에 대해 살펴보자.

엑세스 네트워크는 엔드 시스템과 더 넓은 네트워크 인프라 사이의 중요한 연결 고리 역할을 한다. 엑세스 네트워크는 크게 3가지 유형으로 분류할 수 있다.

Home Access Networks

홈 액세스 네트워크는 주거 환경 내에서 연결과 통신을 가능하게 하는 기술 집합을 말한다. 이 네트워크를 통해 가정 내 디바이스는 인터넷에 연결하고, 데이터를 공유하고, 서로 통신할 수 있다.

Institutional Access Networks

기관용 액세스 네트워크는 조직, 기관 및 비즈니스에서 인터넷에 연결하고 구내 통신을 원활하게 하기 위해 사용하는 네트워킹 인프라와 기술을 말한다. 이런 네트워크는 대규모 운영의 특정 필요와 요구사항을 잘 처리하도록 설계되었다.

Mobile Access Networks

모바일 액세스 네트워크는 스마트폰, 태블릿과 같은 모바일 디바이스가 인터넷에 연결하여 서로 통신할 수 있도록 하는 다양한 기술을 말한다. 이런 기술은 모바일 사용자의 무선 통신 및 데이터 전송을 용이하게 한다.

엔드 사용자와 인터넷 간의 last-mile 연결을 제공하기 위한 다양한 기술과 인프라가 있다. 다음은 가정, 기관, 모바일 액세스 네트워크에서 사용되는 가장 일반적인 액세스 네트워크 기술 몇 가지다.

The Network Core

인터넷의 엣지를 살펴보았으니 이제 인터넷의 엔드 시스템을 상호 연결하는 패킷 라우터와 링크의 그물망인 네트워크 코어에 대해 자세히 살펴보자.

인터넷 코어는 패킷 스위칭 원리에 따라 작동한다. 엔드 호스트는 애플리케이션 메세지를 받아 데이터 덩어리로 나누고, 그 데이터 덩어리를 패킷에 넣은 다음, 그 패킷을 인터넷으로 전송한다. 각 패킷에는 대상 주소가 포함되며 네트워크를 통해 독립적으로 라우팅된다. 이를 통해 많은 통신 세션이 코어 네트워크를 공유할 수 있다.

예를 들어, 웹 서버는 애플리케이션 응답을 패킷으로 나눈다. 서버는 이런 데이터 패킷을 전송하고, 이 데이터 패킷은 인터넷 백본 링크를 통해 라우터에서 라우터로 이동해 목적지, 즉 브라우저를 실행하는 사용자의 노트북에 도달한다. 노트북은 패킷을 재조립해 웹 서버의 응답을 재구성한다.

네트워크 코어 내부에서는 포워딩과 라우팅이라는 두 가지 주요 기능이 수행된다.

Forwarding

포워딩은 도착하는 패킷을 라우터의 입력 링크에서 적절한 라우터 출력 링크로 이동하는 로컬 작업이다. 포워딩은 인터넷에 있는 수백만 개의 라우터 각각 내부에 있는 포워딩 테이블에 의해 제어된다.

패킷이 도착하면 라우터는 내부에서 목적지 주소를 찾아 포워딩 테이블에서 해당 목적지 주소를 조회한다. 그런 다음 들어오는 패킷을 해당 목적지로 연결되는 출력 링크로 전송한다.

Routing

라우터가 패킷을 목적지로 이동할 수 있도록 하는 포워딩 테이블은 라우팅 알고리즘에 의해 생성된다. 라우팅은 패킷이 소스에서 목적지까지 이동하는 전체 경로를 결정하는 글로벌 프로세스다.

인터넷 라우팅 알고리즘은 글로벌 네트워크의 두 지점 사이에서 가장 짧고 효율적인 경로를 계산한다. 알고리즘은 현재 네트워크 상태와 트래픽 흐름을 고려해 최적의 경로를 생성한다. 알고리즘은 적절한 경로를 결정하고 각 라우터의 포워딩 테이블을 다음 홉으로 채워 네트워크 목적지에 효율적으로 도달할 수 있도록 한다.

Network Protocols

네트워크에 연결된 디바이스 간의 통신과 관련된 인터넷상의 모든 활동은 표준 규칙을 제공하는 프로토콜에 의해 관리된다. 프로토콜은 메세지 형식, 메세지 교환 순서, 예상 응답을 정의해 장치 간의 적절한 통신을 가능하게 한다.

하드웨어 수준에서 구현된 프로토콜은 물리적으로 연결된 두 개의 네트워크 인터페이스 카드 간의 비트 전송을 제어한다. 혼잡 제어 프로토콜은 최종 시스템이 패킷을 주고 받는 속도를 조절한다. 라우팅 프로토콜은 각 패킷이 소스와 목적지 사이에서 이동해야 하는 경로를 매핑한다. 프로토콜은 네트워크의 모든 활동을 조정한다.

몇 가지 일반적인 예로는 안정적인 데이터 전송을 지원하는 TCP, 논리적 주소 지정을 위한 IP, 웹 트래픽 구조를 정의하는 HTTP, 파일 전송을 위한 FTP가 있다. 각 프로토콜은 안정적인 데이터 전송, 주소 지정, 라우팅, 다양한 유형의 네트워크 서비스 관리 등 네트워크 내에서 특정 목적을 수행한다.

표준화는 인터넷 엔지니어링 태스크 포스(IETF)에서 이루어지며 이 단체는 RFC라는 기술 표준을 발표한다. 이런 사양을 통해 다양한 하드웨어 및 소프트웨어 구현이 상호 운용성을 달성할 수 있다.

The Internet Protocol Stack (TCP/IP Stack)

네트워크 설계자는 네트워크 프로토콜 설계에 구조를 제공하기 위해 프로토콜과 프로토콜을 구현하는 네트워크 하드웨어 및 소프트웨어를 계층적으로 구성한다. 각 프로토콜은 계층 중 하나에 속한다. 각 계층은 해당 계층 내에서 특정 작업을 수행하고 바로 아래에 있는 계층의 서비스를 사용하는 방식으로 서비스를 제공한다. 다양한 계층의 프로토콜을 한데 모아 프로토콜 스택이라 한다.

TCP/IP 스택이라고도 하는 인터넷 프로토콜 스택은 인터넷과 대부분의 컴퓨터 네트워크를 통한 통신에 사용되는 프로토콜을 표준화하는 개념적 프레임워크다. 컴퓨터와 디바이스가 서로 통신하고 데이터를 교환할 수 있도록 하는 일련의 네트워킹 프로토콜이다. OSI 모델은 7개의 계층으로 구성되어 있는 것으로 유명하지만, TCP/IP 모델은 일반적으로 이를 4개의 계층으로 통합한다.

Application Layer

TCP/IP 스택의 최상위 계층은 최종 사용자 또는 애플리케이션에 직접 네트워크 서비스를 제공하는 애플리케이션 계층이다. 여기에는 HTTP, FTP, SMTP, DNS와 같은 프로토콜이 포함된다. 이 계층은 고수준의 통신 및 사용자 인터페이스를 처리한다.

Transport Layer

전송 계층은 디바이스 간의 종단 간 통신을 담당한다. 이 계층은 서로 다른 기기의 애플리케이션 간에 데이터가 안정적이고 효율적으로 전송되도록 보장한다. 이 계층에서 가장 일반적으로 사용되는 프로토콜은 안정적인 연결 지향 통신을 제공하는 TCP와 더 빠른 연결 없는 통신을 제공하는 UDP다.

Network Layer

네트워크 계층은 IP(인터넷 프로토콜)이 있는 곳이다. 이 계층은 서로 다른 네트워크에 있는 장치 간의 데이터 패킷 주소 지정 및 라우팅을 처리한다. 이 계층은 주로 네트워크 계층이 있는 모든 인터넷 구성 요소에 필수적인 IP 프로토콜을 포함한다. 인터넷의 네트워크 계층에는 데이터그램이 소스와 목적지 간에 이동하는 경로를 결정하는 라우팅 프로토콜도 포함되어 있다.

네트워크 계층에는 IP 프로토콜과 수많은 라우팅 프로토콜이 모두 포함되어 있기 때문에, IP가 인터넷을 하나로 묶는 접착제라는 사실을 크게 반영해 네트워크 계층을 간단히 IP 계층이라고 부르기도 한다. IPv4와 IPv6는 이 계층에서 사용되는 두 가지 주요 IP 버전이다.

Link Layer

네트워크 인터페이스 레이어 또는 데이터 링크 레이어라고도 하는 링크 레이어는 동일한 네트워크 세그먼트에 있는 장치 간의 물리적 연결을 담당한다. MAC 주소와 같은 프로토콜과 하드웨어 주소 지정을 처리한다. 이더넷은 이 계층에서 사용되는 일반적인 프로토콜이다.

이런 계층은 함께 작동해 네트워크 전반에서 통신을 가능하게 한다. 데이터는 발신자 측의 프로토콜 스택 아래로 이동하면서 각 계층에서 캡슐화되고, 수신자 측의 스택 위로 이동하면서 캡슐화가 해제된다.

TCP/IP 모델은 인터넷 통신에 널리 사용되며 현대 인터넷의 기반이 된다.
