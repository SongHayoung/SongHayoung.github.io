vector

벡터는 동적 배열의 구현체이다. 동적 배열이기 때문에 크기가 가변이다.

벡터 공간 활용

벡터를 효율적으로 사용하기 위해선 미리 공간을 할당해두어야 한다. 원소가 가득 찬 벡터의 삽입은 다음과 같이 동작한다.

  1. 벡터의 크기를 확인한다
  2. 만약 가득 찼다면 새로운 2 * N 크기의 백터를 생성한다
  3. 새로운 백터에 기존 데이터를 복사한다
  4. 새로운 백터를 리턴한다

원소가 가득 차 있을 때 지수적으로 증가한 벡터를 생성하고 해당 벡터에 복사가 이루어지기 때문에 미리 공간을 할당해두는것이 유리하다. 할당과 동시에 초기화하거나 resize를 통한 공간을 확보하거나 reserve를 통해 공간을 확보한다. resize는 메모리를 할당하며 원하는 값으로 초기화를 진행하고 reserve는 메모리를 할당만 수행한다. capacity를 넘어서는 할당에는 재할당이 이루어지고 이 때 cost가 비싸다.

벡터의 삽입과 삭제

일반적으로 벡터의 삽입엔 O(1) 삭제는 O(N)이라고 알려져있다. 맞는 말이지만 이는 내부 동작을 잘 이해해야 한다. 벡터의 삽입과 삭제는 어느 위치에 원소를 삽입하냐에 따라 수행 속도가 달라진다. 삽입에 있어서 앞쪽에 삽입한다면 뒤에 원소를 복사하는 작업이 필요하다. 결국 [0]에 삽입하게 된다면 수행시간은 O(N)이다. 삭제도 마찬가지다. [N]의 원소를 삭제하게 된다면 수행시간은 O(1)이다. 벡터의 삽입 삭제는 원소의 위치에 따라 달라진다. 하지만 Big-O Notation으로 평균 수행 시간을 산출하면 O(N/2)가 되고 무시되는 상수값을 제(除)하고 나면 O(N)이 된다.

삽입에는 push_back과 emplace_back을 지원한다. push_back은 다음과 같은 특징을 가진다.

  1. 암시적 생성자 호출
  2. 임시 객체 생성 -> 내부에서 복사 생성자 호출 -> 임시 객체 소멸

emplace_back은 다음과 같은 특징을 가진다.

  1. 명시적 생성자 호출
  2. 내부에서 임시 객체 생성

at()

at 함수는 벡터의 원소를 반환한다. []를 통해서도 반환을 받을 수 있다. 하지만 at원소는 인덱스가 유효한지에 대한 검사를 수행한다.
