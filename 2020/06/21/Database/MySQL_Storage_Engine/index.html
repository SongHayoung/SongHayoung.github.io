[MySQL] Storage Engine

Word count: 172|Reading time: 1 min

MyISAM

MySQL의 Storeage Engine은 크게 MyISAM, InnoDB 두가지로 나뉜다
MyISAM에 대해 간략히 설명하고 넘어가자면 테이블에 항상 Row Count에 대한 정보를 가지고 있기 때문에 SELECT count(*)과 같은 쿼리에서 우수한 성능을 보인다 또한 SELECT 쿼리에서 우수한 성능을 보이며 Full Text Index를 엔진레벨에서 지원하기 때문에 조회 작업이 많은 데이터베이스에 적합한 스토리지 엔진이다 또한 백업이 쉽다는 장점이 있다
앞서 말하기에 MyISAM의 장점만 나열하였는데 단점을 이야기해 보다면 대표적으로 Low Level Rocking이 지원되지 않는다 MyISAM은 Table Locking을 지원하기 때문에 Read 작업이 아닌 다른 작업과 병행되어지는 데이터베이스라면 성능에 대한 문제가 발생한다


InnoDB

InnoDB는 Transcation-Safe하게 테이블을 관리하며 Low Level Locking이 가능하고 커밋, 롤백, 외래키와 같은 다양한 트랜잭션 작업을 지원한다 또한 Clustered Index나 Non-clustered Index를 지원하며 Read 작업이 아닌 다른 작업과 병행되어지는 데이터베이스에서 MyISAM보다 우수한 성능을 가지게 된다


Conclusion

MyISAM

  • 단순한 기능을 제공하기 때문에 읽기에 있어서 속도가 빠르다
  • 무결성이 보장되지 않는다
  • 테이블 단위의 락이 이루어진다
  • 트랜잭션 지원을 하지 않는다

InnoDB

  • 로우 레벨 락을 지원한다
  • 트랜잭션 기능을 지원한다
  • 무결성이 보장된다
  • Insert Delete Update같은 작업에서 속도가 빠르다
