tcp vs udp

  • TCP : 연결지향, 흐름제어, 혼잡제어, 오류 검출, 전이중, 점대점 통신 그래서 멀티캐스트, 브로드캐스트 지원 안함
  • UDP : 비연결지향

3way handshaking 4way handshaking

3way handshaking

  • active open -> passive open : syn
    • active open : syn send
    • passive open : listen
  • passive open -> active open : syn + ack
    • active open : syn send
    • passive open : syn recv
  • active open -> passive open : ack
    • active open : establish
    • passive open : establish

4way handshaking

  • active close -> passive close : fin
    • active close : fin_wait 1
    • passive close : close_wait
  • passive close -> active close : ack
    • active close : fin_wait 2
    • passive close : close_wait
  • passive close -> active close : fin
    • active close : fin_wait 2
    • passive close : last ack
  • active close -> passive close : ack
    • active close : time wait
    • passive close : close

osi 7 layer

  • 물리 : 물리적 연결, utp 케이블
  • 데이터링크 : 네트워크 프레임에 mac주소 부여
  • 네트워크 : ip주소 지정, 경로 탐색
  • 전송 : tcp, udp 사용
  • 세션 : tcp/ip 세션을 만들고 없애고
  • 표현 : 데이터 표현
  • 응용 : 실제 목적지
