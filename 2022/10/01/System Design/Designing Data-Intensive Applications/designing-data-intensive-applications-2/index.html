데이터 모델과 질의 언어

다양한 범용 데이터 모델과 각 모델들의 특징을 알아본다

관계형 모델

  • 데이터는 관계(relation)로 구성
  • 관계는 순서 없는 튜플의 모음
  • 정규화된 구조로 데이터를 저장하고 질의할 때 사용

NoSQL

  • 비 관계형 데이터 베이스로부터 출발해 현재는 Not Only SQL로 해석 한다
  • 대규모 데이터나 높은 쓰기 처리량을 RDBMS보다 쉽게 달성 및 뛰어난 확장성
  • 관계형 모델에서 지원하지 않는 특수 질의
  • 스키마 제한에서 벗어난 풍부한 모델링 가능

문서형 모델

  • 스키마 유연성
  • 데이터 지역성에 기반한 더 나은 성능
    • 역으로 지역성에 기반해서 문서를 큰 형태로 두기에는 취약함
  • 관계형 모델보다 1:N N:M관계에 취약함
  • 상호연결이 적고 모델 독립적으로 사용될 때 우수함

그래프형 모델

  • 정점과 간선으로 이루어짐
  • N:M 관계의 연산에 강력함
    • 모든것이 잠재적으로 관련 있다는 가정
